## SAVE / SHARE / PROFILE — LOCKED DECISIONS

### Auth
- Google OAuth — primary. Magic link — fallback. No passwords. Ever.
- Both methods link to same Supabase account via email match.
- Session persists until explicit sign out — user does NOT re-auth on return visits.
- Both auth methods redirect to `/profile` after completing.
- Google OAuth scope: `email` only — no additional Google permissions ever.
- Service role key never in frontend code — Edge Functions only.

### Data
- `saved_trips` table exists (migration 045 applied).
- NO public read RLS policy on `saved_trips` — security requirement, never add one.
- Share token lookups go through `share-token-lookup` Edge Function only — never direct table query.
- `destination_id` FK uses `ON DELETE SET NULL` — saved trips survive destination deletion.
- Share token is 16-byte base64url (128-bit entropy). Never expose full token in UI.
- 50 saved trips per user hard cap — enforced at application layer before insert.
- `kv_pending_trip` localStorage key has 24hr TTL — discard if older.
- `dest.id` (UUID) is the FK for saved_trips — never use `dest.destination_id` (text code).

### Share flow
- Share link encodes destination slug only — NOT user's personal inputs.
- Recipient sees verdict teaser + blurred sections + CTA to build their own plan.
- Share page is fully public — no auth required to view.
- "Unshare" = delete the plan. No token rotation.

### Edge Functions
- `share-token-lookup` — returns only: `destination_slug`, `destination_name`, `verdict`, `arrival_date`, `departure_date`, `group_type`, `weight_bracket`, `sharer_display_name`. Never returns `user_id` or email.
- `delete-account` — requires authenticated request. Uses service role to delete `auth.users` record. `saved_trips` CASCADE deletes via FK.

## ARCHITECTURE DECISIONS (LOCKED)

| Decision | What | Why |
|---|---|---|
| Data source split | Google Places = venues. Perplexity = context/narrative. | Never ask Perplexity to re-collect what Google Places has. |
| Structured JSON output | All Perplexity prompts return pure JSON | Eliminates 40% null rate from label mismatch. |
| Data availability flag | Every response includes `data_availability.complete` | Drives frontend display logic for sparse destinations. |
| Sparse data = honest info | Never show empty cards | Absence of data tells users something real about the destination. |
| RPC-first | Never direct table queries | RLS on normalized tables requires SECURITY DEFINER on every RPC. |
| SECURITY DEFINER | Mandatory on all RPCs joining normalized tables | Missing this = silent all-nulls bug with no error message. |
| Slug routing | `/destinations/:slug` always | No ID routes. |
| No text fields in modal | Tap-only Trip Planner | Reduces friction. Enforces structure. |
| Claude API never generates facts | No airport codes, wind data, hazards from AI | Mauritius hallucination (SIR vs MRU) made this non-negotiable. |
| Human voice | Anti-AI baseline in all content | Users trust honesty and imperfection over polished AI prose. |
| No public RLS on saved_trips | Share lookups via Edge Function only | Prevents anonymous enumeration of user trip data. |
| dest.id not dest.destination_id | UUID FK for all saved_trips inserts | Text code is not the FK — using it causes silent insert failures. |
